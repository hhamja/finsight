# PRD: FinSight

## 목표
사용자가 은행/카드사 CSV 명세서를 업로드하면 Claude가 자동 분석해, 어디에 얼마를 썼고 어디서 새는지를 즉시 보여준다.

## 사용자
매달 카드/계좌 명세서를 받지만 직접 분석하지 않는(또는 엑셀로 수작업하는) 개인. 가계부 앱의 수동 입력에 지친 사람.

## 핵심 기능
1. 범용 CSV 업로드 → Claude가 컬럼(날짜/금액/가맹점)을 자동 인식해 거래 정규화. **어떻게 읽었는지(매핑)와 신뢰도를 화면에 노출**해 사용자가 검산. (한글 CSV는 CP949/EUC-KR 디코딩 폴백)
2. 분석 대시보드: 카테고리별 지출 분류, 기간별 추이, 중복·구독 누수·이상 거래 탐지, AI 요약·절약 제안. **합계·추이·탐지 등 모든 숫자는 코드가 결정론적으로 계산**(LLM은 라벨링·요약만).
3. 분석 이력 저장 및 월별 비교 — **집계 결과만 저장**(거래 단위·원본 CSV 미저장). 재방문 시 집계 재현, 거래 드릴다운은 재업로드.
4. 구독(Polar): Free → Pro 업그레이드로 심화 분석·이력 해제.
5. 데이터 통제: 사용자가 분석 이력·계정을 직접 삭제(개인정보 파기).

## 사용자 여정 (use case)
- UC1 첫 방문자: 랜딩에서 **샘플 명세서 분석을 로그아웃 상태로 미리 체험**(사전 제작 결과) → 가치 이해 → 가입
- UC2 신규 가입자 (Activation): 첫 명세서 업로드 → 카테고리 + 요약 확인
- UC3 Free 사용자 (Conversion): 잠긴 Pro 카드(추이·이상탐지) → 업그레이드 결제
- UC4 Pro 사용자 (Retention): 매월 업로드 → 전월 대비 추이·이상·절약팁
- UC5 재방문자: 과거 분석 이력 열람
- UC6 구독자 (Billing): Polar 포털에서 구독 취소·변경

## 요금제 (KRW · 한국어)
- **Free**: 단건 명세서 → 상위 카테고리 분류 + AI 요약(총평). 경량 모델. 남용·비용 통제용 **일일 분석 횟수 캡**(Postgres 카운터).
- **Pro (₩9,900/월)**: 단건도 **더 깊게** + 이력으로 **더 자라게**.
  - *1일차 즉시 가치(단건만으로 계산 가능)*: 구독·정기결제 탐지("매달 빠지는 구독 N건, 합계 ₩X"), 중복/이중청구 의심, 전체 카테고리 + 가맹점 드릴다운, 가맹점별 구체 절약 제안, opus 모델 품질.
  - *이력이 쌓이며 자라는 retention 보상*: 월별 추이·전월 대비·"평소 대비 이상". 첫 달엔 빈 차트 대신 "다음 명세서를 올리면 추이가 나타납니다" 기대치 설정.
- 게이팅 축은 **횟수가 아니라 기능 깊이/이력**이다. (명세서는 월 1회라 횟수 제한이 무의미 — Free 캡은 비용 방어용)
- 가격(₩9,900)은 기본값이며 출시 전 조정 가능.

## MVP 제외 사항
- 다계좌/다카드 자동 통합 집계
- 은행 실시간 연동(오픈뱅킹/스크래핑)
- 모바일 앱
- 예산 설정·알림·푸시
- 영수증 OCR

## 법적·프라이버시
- 오픈뱅킹·스크래핑·송금을 하지 않으므로(위 제외) **전자금융업 등록 의무 없음** — 데이터 프라이버시에만 집중.
- 싸고 필수: 계정·이력 삭제 기능(개인정보 파기) + 약관·개인정보처리방침 페이지(랜딩 푸터, MVP는 placeholder 수준). 실서비스 전 법무 검토는 별도.

## 디자인
- 다크모드 고정, 미니멀 "도구" 톤 (마케팅 페이지가 아니라 매월 쓰는 대시보드)
- 무채색 + 포인트 1색(앰버/그린 계열). 보라/인디고 금지
