# Architecture Decision Records

## 철학
MVP 속도 최우선. 외부 의존성 최소화. 작동하는 최소 구현을 선택한다. 인프라(인증·DB·결제·세금)는 직접 만들지 않고 검증된 서비스에 위임한다.

---

### ADR-001: Next.js 15 App Router 채택
**결정**: 풀스택 Next.js(App Router)로 프론트 + API를 한 코드베이스에서 운영.
**이유**: Vercel 배포 즉시성, Server Components로 비밀키 보호, API 라우트로 외부 API 프록시.
**트레이드오프**: 프론트/백 분리 유연성 포기. MVP 단계에선 불필요.

### ADR-002: Supabase로 인증 + DB 일원화
**결정**: 인증과 Postgres를 Supabase 하나로.
**이유**: Auth + DB + RLS를 한 번에, 무료 티어 시작, RLS로 행 단위 보안.
**트레이드오프**: 벤더 종속. 마이그레이션 비용은 MVP 가치 대비 수용.

### ADR-003: Polar로 결제(Merchant of Record)
**결정**: 구독 결제를 Polar로.
**이유**: MoR이라 세금·인보이스·환불을 위임. Next.js SDK로 체크아웃·웹훅 빠름.
**트레이드오프**: Stripe 대비 생태계 작음. 세무 부담 제거 이득이 더 큼.

### ADR-004: 범용 CSV + Claude 자동 컬럼 매핑
**결정**: 특정 은행 포맷을 고정하지 않고, Claude가 임의 CSV의 컬럼을 추론.
**이유**: 한국 은행/카드사 CSV 포맷이 제각각. 포맷 종속 시 커버리지·유지보수 폭증.
**트레이드오프**: LLM 호출 비용·지연. AI가 핵심 가치라 정당.

### ADR-005: 집계 결과만 저장(원본 CSV·거래 단위 미저장)
**결정**: 원본 거래 CSV는 메모리 처리 후 폐기. 저장하는 것은 **집계 JSON만**(카테고리 합계·추이 포인트·상위 가맹점·이상탐지·요약·절약팁). 거래 한 줄씩의 원시 리스트(가맹점·금액)는 저장하지 않는다.
**이유**: 핀테크 프라이버시·데이터 책임 최소화. 집계만 남기면 유출돼도 사용자의 전체 명세서를 복원할 수 없다 — "원본 미저장" 약속이 실제로 성립한다.
**트레이드오프**: 재방문 시 거래 단위 드릴다운 불가(재업로드 필요). 월별 추이·전월 대비·"카테고리 평균 대비 이상"은 집계만으로 계산 가능하므로 핵심 가치는 보존.

### ADR-006: Claude tool-use로 구조화 출력
**결정**: 분석 결과를 tool-use(JSON 스키마 강제)로 받음.
**이유**: 자유 텍스트 파싱은 불안정. 스키마 고정으로 타입 안전.
**트레이드오프**: 프롬프트 복잡도 ↑. 안정성 이득이 큼.

### ADR-007: 계산은 코드, LLM은 라벨링 (역할 분리)
**결정**: Claude는 거래 라벨링(카테고리·가맹점 정규화)·컬럼 매핑 추론·정성 요약/제안 문장만 생성한다. 카테고리 합계·기간 추이·전월 대비 %·중복/구독 탐지 등 **모든 산술·집계는 코드(`lib/`)에서 결정론적으로** 계산한다.
**이유**: 핀테크에서 숫자가 틀리면 신뢰가 0이 된다. LLM은 산술이 부정확하고 비결정적(같은 CSV → 다른 합계)이다. 계산을 코드로 빼면 숫자는 항상 정확·재현 가능하고, 토큰은 라벨링에만 들어 비용·지연도 준다.
**트레이드오프**: `services/claude`와 `lib/aggregate`로 책임이 갈려 파이프라인이 2단계가 됨. 정확성·재현성 이득이 큼.

### ADR-008: 모델 티어링 (Free=경량, Pro=opus)
**결정**: 라벨링은 경량 모델(`claude-haiku-4-5`/`claude-sonnet-4-6`)을 기본으로, Pro 심화 분석에만 `claude-opus-4-8`. 모델은 API 라우트가 tier로 선택해 서비스 래퍼에 주입한다(래퍼는 tier를 모름).
**이유**: 계산을 코드로 뺀 뒤 남은 라벨링은 경량 모델로 충분하다. Free 호출 비용이 급감하고, Pro에 "더 똑똑한 모델"이라는 실질 품질 차별점이 생겨 가격을 정당화한다.
**트레이드오프**: 모델 상수 2개·분기 1개 추가. 비용 절감과 Pro 차별화 이득이 큼.

### ADR-009: 매핑 투명성 + confidence 게이트
**결정**: tool-use 출력에 컬럼 매핑(`mapping`: 어느 컬럼을 날짜/금액/가맹점으로 읽었는지)과 `confidence`를 포함한다. 결과 화면 상단에 매핑 요약과 총건수·합계를 노출해 사용자가 눈으로 검산하게 한다. confidence가 낮거나 합계가 비정상이면 분석을 강행하지 않고 "읽지 못했다"고 안내한다.
**이유**: 범용 CSV 자동 매핑(ADR-004)은 "에러 없이 그럴듯하게 틀린" 대시보드를 만들 수 있다. 핀테크에서 조용한 오답은 명시적 실패보다 나쁘다.
**트레이드오프**: 스키마·UI에 필드 추가. 신뢰가 제품의 전부라 정당.
